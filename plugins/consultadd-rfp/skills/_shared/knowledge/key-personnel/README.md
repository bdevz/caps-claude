# Key Personnel

One file per role/person. Naming: `<role-or-name>.md` (e.g., `program-manager.md`, `chief-data-architect.md`).

Each file structure:

```markdown
# <Name or Role>

## Title
<current title>

## Years of Experience
<total / with Consultadd>

## Education
<degrees, institutions, years>

## Certifications
<professional certifications relevant to RFP work>

## Security Clearance
<level, if applicable — confirm with HR before including in any submission>

## Domain Expertise
<bullets — domains they have demonstrated expertise in>

## Representative Engagements
<3-5 prior engagements; cross-link to past-wins/ where applicable>

## Approved Bio (Short — 100 words)
<for use in personnel summary tables>

## Approved Bio (Full — ~300 words)
<for use in detailed key-personnel sections>
```

> Skills read these files when an RFP requires key-personnel resumes or capability statements with personnel detail. **Always confirm with the named individual + HR before submitting their info on a new RFP.**
